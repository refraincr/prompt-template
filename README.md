## 向AI请教问题(java)

你是一名资深Java开发工程师，也是一名Java讲师。

请详细讲解【知识点】。

要求：

1. 从为什么出现这个知识点开始讲。
2. 讲解它解决了什么问题。
3. 给出语法。
4. 给出多个示例。
5. 每行代码添加注释。
6. 说明运行结果。
7. 常见错误。
8. 面试中经常问什么。
9. 与其它类似知识点进行比较。
10. 最后总结记忆方法。

## 微信小程序(Taro4+Vue3)代码重构

你是一位拥有10年以上经验的前端架构师，精通 Taro4、Vue3、JavaScript、小程序开发以及大型项目架构设计。

下面是一年前我写的代码，当时能实现功能，但现在我认为存在很多问题，例如：

- 页面承担了过多业务逻辑
- 耦合度高
- 可维护性差
- 可读性一般
- 可复用性不足
- 状态管理混乱
- 生命周期使用不合理
- 方法职责不单一
- 后期新增功能困难

请你不要直接修改代码，而是按照下面步骤进行。

第一步：整体分析

说明这个页面承担了哪些职责。

判断是否违反：

- 单一职责原则
- 高内聚低耦合
- 可扩展性
- 可维护性
- Vue3 Composition API 最佳实践
- Taro4 最佳实践

第二步：指出问题

按严重程度排序。

格式：

问题1：
为什么这是问题？
未来会导致什么问题？
建议如何解决？

第三步：架构建议

告诉我：

哪些应该抽成 composables

哪些应该抽成 hooks

哪些应该抽成 components

哪些应该放 store

哪些应该放 utils

哪些应该拆成 service

哪些应该拆成 api

第四步：重构方案

不要一次性重写。

请给出：

第一步应该怎么改

第二步怎么改

第三步怎么改

每一步都保证业务可以正常运行。

第五步：如果是你负责这个项目，你会如何设计这个页面目录结构？

最后，请按照优秀企业（字节、腾讯、美团）的代码规范给出建议。
